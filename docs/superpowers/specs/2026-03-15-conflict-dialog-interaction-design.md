# ConflictDialog 交互优化设计文档

> 创建日期: 2026-03-15

## 背景与目标

### 问题
当前 ConflictDialog 每行都有一个下拉选择框来设置冲突处理策略（跳过/保留两者/覆盖），当冲突项较多时，逐个设置效率低下。

### 目标
- 支持批量选择多个冲突项
- 支持拖选和点击混合的多选方式
- 选中文件夹时自动包含所有子项
- 简化界面，提升操作效率

## 交互模式

**选中优先模式**：先选文件 → 再设策略

1. 默认所有冲突项的策略为"跳过"
2. 用户通过点击/拖选选中文件
3. 点击策略按钮（跳过/保留两者/覆盖）对选中项批量设置

## 核心功能

### 1. 多选交互

| 操作方式 | 行为 |
|---------|------|
| 点击 | 切换单项选中状态 |
| Shift + 点击 | 范围选择（从上一个选中项到当前项） |
| Ctrl + 点击 | 增选/取消选中 |
| 鼠标拖动 | 框选多个文件 |

### 2. 层级展示

- 使用**缩进 + 连接线**展示文件层级关系
- 选中文件夹时**自动选中所有子孙项**
- 取消选中文件夹时**自动取消所有子孙项**

### 3. 信息展示

- 默认只显示：文件图标 + 文件名 + 文件夹标签 + 策略标签
- 源文件/目标文件详细信息（大小、修改时间）通过**悬停 Tooltip** 显示

### 4. 操作栏设计

```
┌────────────────────────────────────────────────────────────┐
│ [☐] 全选/已选中 N 项      [跳过] [保留两者] [覆盖]        │
└────────────────────────────────────────────────────────────┘
```

## 数据结构变更

### ConflictItem 类型扩展

```typescript
// src/types/file.ts

interface ConflictItem {
  id: number;           // 新增：唯一标识
  sourceKey: string;
  targetKey: string;
  isFolder: boolean;
  depth: number;        // 新增：层级深度 (0 = 顶层)
  parent: number | null; // 新增：父节点 ID
  sourceInfo?: {
    size: number;
    lastModified: string;
  };
  targetInfo?: {
    size: number;
    lastModified: string;
  };
}
```

## 实现步骤

### Step 1: 数据结构扩展
- 修改 `ConflictItem` 接口，添加 `id`、`depth`、`parent` 字段
- 修改后端 `server/index.js` 中的冲突检测逻辑，构建层级数据

### Step 2: 组件状态管理
- 新增 `selectedItems: Set<number>` 状态
- 新增 `getItemWithDescendants()` 辅助函数获取子孙节点

### Step 3: 多选交互实现
- 实现点击选中/取消逻辑
- 实现 Shift 范围选择
- 实现 Ctrl 增选
- 实现拖选框选

### Step 4: 层级展示
- 根据深度计算缩进
- 绘制连接线
- 实现文件夹级联选择

### Step 5: Tooltip 实现
- 悬停显示源/目标文件详细信息
- 使用 CSS group-hover 或 shadcn/ui Tooltip 组件

### Step 6: 操作栏重构
- 全选复选框
- 策略按钮（对选中项生效）
- 选中计数显示
